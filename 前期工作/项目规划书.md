# 智能校园自习室预约管理平台 — 项目规划书（大二修订版）

---

## 一、技术栈（务实降级版）

| 层级 | 技术 | 为什么选它 |
|------|------|------------|
| 后端框架 | **Spring Boot 2.7.x** | B站教程最多，遇到bug好搜 |
| ORM | **MyBatis-Plus** | 单表CRUD几乎不用写SQL，分页一行代码 |
| 数据库 | **MySQL 8.0** | 课上学过，直接用 |
| 认证 | **拦截器 + Session** | 别碰Spring Security，写个HandlerInterceptor就够了 |
| 定时任务 | **@Scheduled** | Spring自带注解，释放超时座位用 |
| 接口文档 | **Knife4j** | 自动生成，前后端联调不用吵架 |
| 前端框架 | **Vue 3 (Options API)** | 大二用Options API比Composition API好理解 |
| UI组件库 | **Element Plus** | 表格/表单/弹窗全是现成的，直接复制官网示例 |
| 图表 | **ECharts 5** | 热力图、柱状图copy示例改数据就行 |
| 前端构建 | **Vite** | npm run dev 秒启动 |

## 五、数据库表设计（6张表，够用不多余）

```sql
-- 1. 校区表
campus (id, name, create_time)

-- 2. 楼栋表
building (id, campus_id, name, create_time)

-- 3. 楼层表
floor (id, building_id, floor_number, create_time)

-- 4. 自习室表
room (id, floor_id, name, total_rows, total_cols, open_time, close_time, create_time)

-- 5. 座位表（核心！）
seat (id, room_id, row_num, col_num, status, create_time)
-- status: 0空闲 1已占用 2维修中

-- 6. 预约表（核心！）
reservation (id, user_id, seat_id, room_id, start_time, end_time,
              status, sign_time, create_time)
-- status: 0已预约 1已签到 2已取消 3已超时 4已完成

-- 7. 用户表
user (id, username, password, real_name, role, create_time)
-- role: STUDENT / ADMIN

-- 8. 黑名单表
blacklist (id, user_id, miss_count, banned_until, create_time)
```

> B同学：先把这8张表的SQL写好，全员评审通过后再动手写代码。

## 七、降低难度的开发策略

| 问题 | 大二友好做法 |
|------|--------------|
| 不知道怎么写Controller | 一个表一个Controller，增删改查全是复制粘贴改表名 |
| 座位状态并发冲突 | 先不管！用数据库行锁（SELECT ... FOR UPDATE）一句话的事 |
| 前端不会写 | Element Plus官网每个组件都有完整代码示例，直接copy改字段名 |
| ECharts不会配置 | 官网示例 → 复制option对象 → 只改data数据 → 完事 |
| 接口联调对不上 | Knife4j生成文档网页，前端照着点，不用猜字段名 |
| 定时任务怎么写 | @Scheduled(cron="0 */5 * * * ?") 就是每5分钟执行一次 |
| 密码怎么存 | MD5加盐，一个工具类10行代码，别想BCrypt |
