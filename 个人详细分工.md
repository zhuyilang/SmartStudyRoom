## 👤 C — 后端：预约模块 + 座位管控 + 黑名单

### 第5天：预约接口框架
- [ ] 写 `entity/Reservation.java`
  - 字段：userId、seatId、roomId、startTime(LocalDateTime)、endTime(LocalDateTime)
  - status：0已预约/1已签到/2已取消/3已超时/4已完成
  - signTime(LocalDateTime)
- [ ] 写 `mapper/ReservationMapper.java`
  - 自定义查询：`countTodayReservations(userId)` 今日预约次数
  - 自定义查询：`findConflict(seatId, startTime, endTime)` 时间段冲突检查
- [ ] 写 `service/ReservationService.java` + `impl`

### 第6天：预约核心逻辑
- [ ] 实现 `reserve(userId, seatId, startTime, endTime)`：
  1. 检查用户是否在黑名单（查blacklist表，bannedUntil > 现在）
  2. 检查今日预约次数是否已达上限（默认3次）
  3. 检查该座位在 startTime~endTime 是否已被预约
  4. 检查 startTime 必须 < endTime
  5. 插入预约记录（status=已预约）
  6. 更新座位状态为"已占用"
  7. 以上全部加 `@Transactional`
- [ ] 实现 `cancel(reservationId, userId)`：
  1. 只能取消自己的、状态为"已预约"的记录
  2. 状态改为"已取消"
  3. 座位状态改回"空闲"
- [ ] 写 `controller/ReservationController.java`
  - `POST /api/reservation` 创建预约
  - `PUT /api/reservation/{id}/cancel` 取消预约
  - `GET /api/reservation/my` 我的预约列表（可按状态筛选）
  - `GET /api/reservation/{id}` 预约详情

### 第7天：预约约束完善
- [ ] 单日预约次数限制逻辑：`countTodayReservations(userId)` 超过上限抛异常
- [ ] 时间段冲突检测：SQL → `SELECT COUNT(*) FROM reservation WHERE seat_id=? AND status IN (0,1) AND NOT (end_time <= ? OR start_time >= ?)`
- [ ] 自习室开放时间校验：预约时间必须在 room.openTime ~ room.closeTime 范围内
- [ ] 不允许跨天预约（简化处理，endTime日期必须等于startTime日期）

### 第8天：签到 + 座位状态
- [ ] 实现 `signIn(reservationId, userId)`：
  1. 只能签到自己的、状态为"已预约"的记录
  2. 当前时间必须在 startTime 前后15分钟内
  3. 更新 signTime = 当前时间
  4. 更新 status = "已签到"
  5. 座位状态保持"已占用"
- [ ] 补充 `ReservationController`
  - `PUT /api/reservation/{id}/signIn` 签到
- [ ] 写 `service/SeatStatusService.java`（座位状态统一管理）
  - `occupySeat(seatId)` → 状态改"已占用"
  - `releaseSeat(seatId)` → 状态改"空闲"
  - 所有改座位状态的地方都通过这个Service

### 第9天：黑名单逻辑
- [ ] 写 `entity/Blacklist.java`
  - 字段：userId、missCount（累计爽约次数）、bannedUntil（禁止预约到何时）
- [ ] 写 `mapper/BlacklistMapper.java`
- [ ] 写 `service/BlacklistService.java` + `impl`
  - `recordMiss(userId)`：
    1. 查blacklist表，若无记录则插入 missCount=1
    2. 若有记录，missCount+1
    3. 若missCount >= 3，设置 bannedUntil = 当前时间 + 3天
  - `isBanned(userId)`：查 bannedUntil > 当前时间
  - `removeBan(userId)`：管理员手动解除
- [ ] 写 `controller/BlacklistController.java`
  - `GET /api/blacklist/list` 黑名单列表（管理员）
  - `DELETE /api/blacklist/{userId}` 解除黑名单（管理员）
- [ ] 在A的定时任务中，超时释放时调用 `recordMiss(userId)`

### 第11天：看板数据接口
- [ ] 写 `controller/DashboardController.java`
  - `GET /api/dashboard/seatStatus` 返回所有自习室的座位占用统计
    - 返回格式：`[{roomId, roomName, buildingName, totalSeats, occupiedSeats, freeSeats}]`
  - `GET /api/dashboard/roomDetail/{roomId}` 返回单个房间座位网格
    - 返回格式：`{roomId, rows, cols, seats:[{row,col,status}]}`
